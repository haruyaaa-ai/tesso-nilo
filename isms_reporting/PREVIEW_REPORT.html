ABSTRAK

Proyek ini membahas penerapan Sistem Manajemen Keamanan Informasi (ISMS) berbasis standar internasional ISO/IEC 27001:2022 pada aplikasi web "Tesso Nilo". Latar belakang proyek ini didasari oleh meningkatnya ancaman siber terhadap data pengguna dan transaksi reservasi online. Tujuan utama penerapan ISMS adalah untuk menjaga kerahasiaan (Confidentiality), integritas (Integrity), dan ketersediaan (Availability) aset informasi. Hasil penelitian menunjukkan bahwa implementasi kontrol teknis mampu menurunkan level risiko secara signifikan.

BAB I – PENDAHULUAN

Sistem Informasi Tesso Nilo mengelola data sensitif termasuk identitas pengguna dan data transaksi reservasi. Kebocoran data ini dapat berakibat buruk bagi reputasi organisasi dan kerugian finansial bagi pengguna.

BAB IV – ANALISIS RISIKO KEAMANAN INFORMASI

4.1 Identifikasi Ancaman: Ancaman mencakup serangan siber (SQLi, Brute Force), kegagalan teknis server, kesalahan manusia oleh admin, dan manipulasi data pembayaran oleh pihak luar.

4.2 Identifikasi Kerentanan: Kelemahan ditemukan pada sanitasi input, belum adanya pembatasan percobaan login, dan penggunaan protokol non-enkripsi di localhost.

4.3 Dampak Risiko: Dampak diukur dari hilangnya kerahasiaan data pengguna, integritas transaksi, dan ketersediaan sistem saat dibutuhkan.

4.4 Penilaian Level Risiko: Level diukur dari skala Low (dapat diterima) hingga High (membutuhkan penanganan segera).

4.5 Tabel Risiko Keamanan Informasi

Aset Ancaman Level Mitigasi Utama
Database User Kebocoran Data High Hashing BCrypt, Prepared Statements
Sistem Web Brute Force Medium Login Delay, Password Policy
Data Tiket Manipulasi Data High Manual Verification, API Security
Server DDoS / Down Medium Firewall & Rate Limiting
Akun Admin Phishing High MFA & Security Awareness
File Upload Web Shell (RCE) High MIME Validation & File Renaming

BAB VI – STATEMENT OF APPLICABILITY (SoA)

6.1 Konsep Statement of Applicability: SoA adalah dokumen wajib ISO 27001 yang merincikan pemilihan kontrol dari Annex A, memberikan justifikasi pemilihan, dan status implementasi untuk mengelola risiko informasi yang telah teridentifikasi.

6.2 Tabel SoA Annex A (ISO/IEC 27001:2022)

Kode Nama Kontrol Status Alasan & Bukti
A.5.15 Control of Access Implemented Justifikasi: RBAC Policy. Bukti: Check session role in login.php.
A.8.24 Cryptography Implemented Justifikasi: Password Protection. Bukti: password_hash() BCrypt.
A.8.28 Secure Coding Implemented Justifikasi: SQLi Prevention. Bukti: Prepared Statements (MySQLi).
A.5.7 Threat Intel Implemented Justifikasi: Log Analysis. Bukti: debug_login.log.
A.8.12 Data Leakage Prevention Implemented Justifikasi: Error masking. Bukti: display_errors = 0.
A.8.20 Network Security Not Implemented Justifikasi: SSL/HTTPS Migration pending for enterprise deployment.

BAB VII – IMPLEMENTASI

Kriptografi: Password dienkripsi menggunakan algoritma BCrypt sebelum disimpan ke database.

Akses Kontrol: Role-based access control (RBAC) membedakan hak akses antara Administrator dan Customer.

BAB VIII – HASIL AUDIT INTERNAL ISMS

8.1 Tujuan & Ruang Lingkup: Memverifikasi kepatuhan implementasi terhadap kontrol ISO 27001:2022 pada seluruh modul aplikasi dan database Tesso Nilo.

8.2 Metodologi: Menggunakan teknik review dokumen kebijakan, observasi operasional dashboard, dan inspeksi kode program (Walkthrough).

8.3 Tabel Temuan Audit Internal

No Kontrol ISO Temuan Audit Kategori Rekomendasi
1 A.5.15 (Access Control) Pemisahan role admin & user sudah berjalan. Conformity Sertakan audit user berkala.
2 A.8.24 (Cryptography) Password aman dengan BCrypt hashing. Conformity Pertahankan salt kuat.
3 A.8.20 (Network Security) Koneksi belum menggunakan HTTPS/SSL. Nonconformity Migrasi ke SSL/TLS.
4 A.5.7 (Threat Intel) Log belum mencatat IP detail. OFI Tambahkan IP tracking log.

8.4 Analisis Ketidaksesuaian: Akar penyebab (root cause) belum digunakannya SSL adalah batasan lingkungan pengembangan lokal (XAMPP). Dampaknya adalah kerentanan terhadap serangan MitM.

BAB XI – ANALISIS DAN EVALUASI

11.1 Evaluasi Efektivitas: Kontrol kriptografi (BCrypt) dan Akses (RBAC) dinilai Sangat Efektif. Keamanan jaringan dinilai Perlu Peningkatan karena transisi ke SSL/HTTPS masih tertunda.

11.2 Kelebihan & Kelemahan: Kelebihan pada secure coding dan pencegahan SQLi. Kelemahan pada ketiadaan 2FA untuk admin dan detail log IP.

BAB XII – KESIMPULAN & REKOMENDASI

12.1 Kesimpulan: Implementasi ISMS ISO 27001:2022 telah berhasil menguatkan postur keamanan aplikasi Tesso Nilo secara signifikan di sisi teknis.

12.2 Rekomendasi: Segera aktifkan SSL/TLS, perinci audit log, dan lakukan pengesahan kebijakan formal.

12.3 Rencana Selanjutnya: Integrasi autentikasi dua faktor (2FA) dan sistem automated vulnerability assessment.
